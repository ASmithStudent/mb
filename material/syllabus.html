What to expect from this course

Welcome! This page describes how this course is structured and how students can get the most out of it. I'd use the term "syllabus" if that didn't sound so formal! Let me (the instructor) know if you have any questions or concerns.

Goal

The goal of this course is to learn the basics of computer programming by working in groups to create a game or application using the languages of the web: HTML, CSS, and JavaScript. Groups will present their work on Demo Day at the end of the semester.

Student learning outcomes

  1. Describe the basics of HTML: tags, attributes, and HTML document structure.

  2. Write a simple website in HTML including links and images.

  3. Explain the purpose of CSS, CSS selectors, CSS properties, and CSS classes.

  4. Write a simple CSS stylesheet with multiple selectors and attributes.

  5. Explain the purpose of fundamental computer programming concepts: variables, conditionals, functions, objects, arrays, and loops.

  6. Apply fundamental programming concepts to the creation of a game or application using JavaScript.

  7. Soft skills: planning, focus, effective group work, using the internet to resolve programming problems, developing a positive growth mindset.

Prerequisites

This course assumes no prior knowledge other than internet browsing/computer skills, keyboard and mouse skills, the ability to follow detailed instructions, and the ability to apply logic to solve problems individually and collaboratively.

How this course works

The course is 12 weeks long with four hours of in-class time per week. The first half of the course will provide a foundation in web languages, and the second half will involve application of the concepts through project work. That sounds like a lot of time, but it boils down to a mere 24 hours to learn programming and 24 hours to create your project! Using only this in-class time will give you decent exposure to programming, but to truly move forward, students should spend time outside of class exploring as well.

The entire session is group-based. I do realize group work can be painful for many (myself included!), but there's no avoiding it in life, so let's dive right in, get comfortable with it, and make it fun!

The language concepts during the first half of the course are presented in modules. Each module's content is roughly a class period, but because students learn at their own pace, students should feel welcome to spend more or less time on a module as necessary.

I will work to facilitate the groups, ensuring that students are engaging the content and their classmates effectively. I will also offer lectures when necessary to clear up points of confusion, explain general concepts, and keep the class moving forward. There will be adult mentors, volunteers and other Mission Bit staff present to assist in students' learning experience.

Classroom culture

My main job as the class instructor, beyond simply facilitating the course content, is to ensure that everyone in class feels safe, comfortable and welcome. I encourage freedom of expression and creativity, realizing that sometimes in that process, misunderstandings can happen and feelings can be hurt. I ask that all members of the class community work together to resolve any issues that may arise, and strive to be respectful and inclusive in their words and in their creations.

Guidelines for success

  • Be present. Mission Bit allows for no more than three absences; however, so much material will be covered in each class that it's very easy to fall behind missing one or two. If you have any concerns about attendance, bring them to the instructor's attention today and we'll work it out!

  • Focus is the key to learning stuff, whatever it may be. We all struggle with constant distractions from games, texting, 10-hour Game of Thrones marathons, etc. These things seem fun in the short run, but in the long run, the folks who get great at something make time for the important stuff. It's a bit ironic that consuming computer-based content is one of the biggest barriers to learning how to make computer-based content.

  • Don't worry if you get confused, frustrated, or stuck on a problem. These are completely normal parts of the learning process and happen to everyone.

  • Type in all of the examples by hand instead of copying and pasting. This helps comprehension enormously.

  • Help your friends; teaching and collaborating are great ways to learn. Be supportive, encouraging, and contribute to the class community.

  • Ask questions when you get stuck.

  • Most importantly:

    Have Fun!
